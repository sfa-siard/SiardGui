SIARD Suite 2.2 - Introduction à l'archivage de bases de données

Ceci est un bref aperçu sur la manière d'archiver des bases de données avec SIARD Suite. Des thèmes organisationnels étant pertinents pour l'archivage de bases de données sont également abordés.

  1. Clarifier quelles parties de la base de données doivent être archivées. Si nécessaire, les personnes responsables doivent être consultées, par exemple le service d'évaluation de l'archive.
  2. Préparer la base de données pour l'archivage: Créer un nouvel utilisateur de la base de données qui ne doit détenir que les droits de lecture sur l'objet à archiver. Si nécessaire, créer une copie de la base de données (ou des parties à utiliser) ou créer des vues. Durant l'archivage, la base de données ne doit subir aucune modification sinon il échouera. Ne jamais archiver depuis un système live.
  3. Télécharger la base de données avec SIARD Suite.
  4. Contrôle de qualité: Vérifier le fichier SIARD pour garantir que tous les objets nécessaires ont été archivés. Effectuer des échantillonnages afin de contrôler la qualité des données.
  5. Contrôle de qualité étendu: Charger le fichier SIARD dans un système de base de données. Effectuer quelques requêtes dans la base de données originale et dans la base de données archivée et comparer les résultats.
  6. Compléter le fichier SIARD avec les données primaires.
  7. Définir quelle documentation externe doit être archivée avec le fichier SIARD afin d'en garantir la compréhensibilité (par ex. tableau de codes, documentation système, diagramme de relations, ...).
