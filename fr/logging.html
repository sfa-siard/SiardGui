SIARD Suite 2.2 - Protocole

Si un problème surgit lors de l'utilisation de SIARD Suite, il est utile d'enregistrer un protocole d'exécution. Un tel protocole (ou - pour des raisons de sécurité - un extrait de celui-ci) devrait être joint au message d'erreur adressé à la maintenance.

Pour enregistrer un protocole, il faut procéder comme suit:

Dans le répertoire etc de l'installation SIARD, on trouve le ficher logging.properties. L'enregistrement du protocole avec le niveau WARNING sur la console et tous les niveaux sont redirigés dans le fichier log %t/siard2%u.log. Le %t représente ici le répertoire temporaire (valeur des variables d'environnement TEMP ou TMP) et le %u un numéro de séquence Sous Windows, cela donne normalement C:\Users\<user>\AppData\Local\Temp\siard20.log).

Le niveau global de log se trouve initialement dans INFO. Un logging plus détaillé peut être configuré en modifiant la ligne
.level=INFO
en
.level=ALL.
Un logging détaillé ralentit l'exécution.
