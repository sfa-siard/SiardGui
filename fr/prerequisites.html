SIARD Suite 2.2 - Conditions d'utilisation

La condition pour l'utilisation de SIARD Suite est la présence d'un JAVA Runtime Environment (JRE). Le téléchargement depuis ou le chargement dans un système de base de données exige une infrastructure correspondante.

Haut de page

JAVA

Les utilisateurs peuvent soit télécharger une version de SIARD Suite qui inclut un JRE, soit installer un JRE sur leur système d'exploitation. JAVA peut être librement obtenu sur le site http://www.java.com/ (JRE – JAVA Runtime Environment) ou sur le site http://www.oracle.com/technetwork/java/javase/downloads/index.html (JDK - JAVA Development Kit). Remarque: SIARD Suite utilise des fonctionnalités de JavaFX qui n'a été intégré que dans les versions 8-10 de la distribution JAVA.

Pour savoir si une version appropriée de JAVA (8-10) est disponible sous Windows, procédez comme suit: Dans le menu Démarrage, taper la commande "cmd" et saisir "java -version" dans la fenêtre de commande.

Architecture (32-bit/64-bit)

Dans les versions précédentes de SIARD Suite, il y avait une dépendance à ODBC. Cela signifiait que seule la version 32-bit de JAVA pouvait être utilisée pour accéder à des bases de données MS Access. SIARD Suite 2.2 n'utilise plus ODBC pour la connexion à des bases de données MS Access. On peut à présent accéder à ces bases de données aussi sur d'autres plateformes (par ex. LINUX) et avec JAVA 64-bit. Il est recommandé de choisir l'architecture JAVA en fonction du système d'exploitation.

bases de données

SIARD Suite 2.2, dans la livraison actuelle, accepte les systèmes de bases de données suivants:

  • MS Access 2007 ou plus récent
  • DB/2 8 ou plus récent
  • MySQL (ou MariaDB) 5.5 ou plus récent
  • Oracle 10 ou plus récent
  • PostgreSQL 11 ou plus récent
  • SQL Server 2012 ou plus récent

D'autres systèmes de bases de données pourront être intégrés par la suite. Les pilotes JDBC du constructeur de bases de données ne sont normalement pas conformes à SQL:2008. La plupart n'intègre même pas correctement le standard JDBC 4 dans les domaines des métadonnées et de types de données complexes. C'est pourquoi il faut développer une enveloppe JDBC au moins conforme aux standards it pour chaque système de base de données, comme exigé pour utiliser SIARD Suite.
