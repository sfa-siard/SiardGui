Download of the Sources

Prerequisites

In order to make use of the sources of the project SiardGui the following must be installed on the machine of the developer:

  • JAVA JDK 1.8 or higher,
  • Git 2.7 or higher
  • Ant 1.9.3 or higher
  • Eclipse (recommended)

Downloading the Sources

Executing 

git clone https://github.com/sfa-siard/SiardGui.git

will create the folder SiardGui with a local .git repository and the project sources.

When the project is opened in Eclipse many compilation errors are displayed. This is caused by the fact, that some JAVA source files are to be generated automatically by running the generate task of build.xml. However, some preparation for the build is needed first (v. Building the Binaries).

Folder Structure

The SIARD projects all share a similar folder structure:

bin/
executable commands and shell scripts
dist/
binaries resulting from the build of the project (generated by the build)
doc/
documentation of the project
doc/javadoc/
javadoc documentation of published APIs of the project
doc/licenses/
license files for third-party code
doc/manual/
manuals documenting the project
doc/manual/users/
documentation for users of the binaries (in HTML5 format)
doc/manual/developers/
documentation for developers of the structure of the project, its position in the SIARD Architecture, and on how to build it (in HTML5 format)
doc/manual/assets/
common files for all manual pages
doc/specifications/
specifications important for the project
etc/
files for run-time configuration (e.g. logging.properties)
lib/
binaries of other projects and third-party code
src/
source code of the project
test/
source code of (JUnit) tests of the project
testfiles/
test files needed by the tests
tmp/
folder for temporary files (test output, logging output) (generated by the build)
var/
other files needed for the project
build.properties.template
template for local build properties needed for configuring the build process
build.properties
properties describing the local environment of the developer (must be created manually and is never checked in!)
build.xml
Ant script for building the binaries
LICENSE.txt
license of the project
README.txt
very short description of the project
RELEASE.txt
documentation of release changes
.gitignore
single and global list of folders and files in the project which should not be committed to the repository
