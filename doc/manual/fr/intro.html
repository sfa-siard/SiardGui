SIARD Suite 2.1 - Introduction

Introduction

Les Archives fédérales suisses ont développé le format SIARD et l'application SIARD Suite. SIARD (Software-Independent Archival of Relational Databases) sert à archiver à long terme des contenus de bases de données relationnelles.

Sur mandat des Archives fédérales suisses, Enter AG a développé en 2007 le format SIARD et SIARD Suite 1.0, et en 2016-2018 la version présente SIARD Suite 2.1.

En collaboration avec le projet européen E-ARK et laKOST, la version 2.1 du format SIARD a été spécifiée. Comme déjà pour la version 1.0, le format SIARD 2.1 a été homologué comme standard eCH-0165 par l'association eCH E-Government Standards.

SIARD Suite 2.1 est l'implémentation de référence pour l'archivage de bases de données relationnelles en format SIARD 2.1 standardisé.

Le présent document constitue le manuel de SIARD Suite 2.1.

Nouveautés de la version 2.1

Les nouveautés les plus importantes du format SIARD 2.1 comparé à la version 1.0 sont

  • la conformité avec SQL:2008, en particulier le support de nouveaux types de données (DISTINCT, UDT, ARRAY)
  • possibilité de délocaliser de gros objets dans des fichiers externes
  • support de la compression reversible "deflate" des données SIARD

Les fichiers SIARD comformes au format 1.0 peuvent être lus par les programmes de SIARD Suite 2.1. Dès que l'on effectue des modifications et qu'on les enregistre, ces dernières sont converties au format 2.1.
