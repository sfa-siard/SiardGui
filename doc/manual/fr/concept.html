SIARD Suite 2.2 - Concept

Les Archives fédérales suisses sont tenues d'archiver les documents de l'administration fédérale, indépendamment du porteur d'information. La problématique de l'archivage à long terme de bases de données relationnelles doit ainsi être résolu.

Depuis 2007, le format SIARD Format sert de format de normalisation aux Archives fédérales suisses - ainsi qu'à beaucoup d'autres archives dans le monde entier - pour la conservation à long terme de bases de données relationnelles.

Afin d'en assurer la disponibilité à long terme, les bases de données sont conservées de manière conforme à SQL:2008. les contenus sont conservés dans une collection de fichiers SML. Etant donné que le format d'archivage pour bases de données se base sur ces deux standards ISO, on lui attribue une compréhension de plus longue durée.

Une exigence importante pour les données archivées en format SIARD est que le contenu de la base de données ait "caractère de documents". Cela signifie que les contenus des tableaux archivés contiennent des données commerciales que la base de données exploite et qui sont compréhensibles, quelque soit l'application frontend. Avec SIARD Suite, aucun code de programme exécutable ni des objets ne sont archivés, mais uniquement des données commerciales dans les tableaux des bases de données. Cette approche est largement décrite par Hartwig Thomas dans le document "Long-term Preservation of Relational Databases, What needs to be preserved how?".

Dans le format SIARD, la définition du schéma d'une base de données archivée est enregistrée dans des fichiers XML conformes à SQL:2008 alors que que la documentation des tableaux et des champs ainsi que les données sont enregistrées dans des fichiers XML. Afin que ces derniers ne gonflent pas exagérément, les données BLOB et CLOB (Binar4y Large OBjects et Character Large OBjects) sont enregistrées dans des fichiers (binaires) séparés qui sont référencés dans les fichiers XML.

La structure des formats SIARD n'est pas expliquée plus en détail dans ce manuel car elle est décrite dans un autre document qui est livré avec SIARD Suite. En 2013, le format SIARD a été reconnu comme standard eCH. Depuis 2017, c'est la version 2.1 du format SIARD qui est disponible comme standard eCH-0165.
