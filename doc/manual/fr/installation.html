SIARD Suite 2.1 - Installation

Les menus ci-après concernent l'installation:

  • Installer: installation de la copie actuellement en fonction de SIARD Suite,
  • Désinstaller: désinstallation de la copie actuellement installée de SIARD Suite.

Haut de page

Que signifie Installation?

SIARD Suite peut être utilisé sur toutes les plates-formes où JAVA est disponible. C'est la raison pour laquelle l'installation n'est pas réalisé dans un programme d'installation spécifique à une plate-forme (par ex. setup.exe) mais dans SIARD Suite.

L'installation consiste à copier les fichiers de programmes par ex. depuis un support de données ou depuis un répertoire temporaire dans un répertoire fixe (réseau ou disque dur) et à enregistrer quelques paramètres dans le répertoire propre à l'utilisateur qui garde en mémoire l'état des appels du programme (par ex. C:\Users\<User>\.java\siard_suite-2.1.properties).

Il est ainsi possible d'exécuter SIARD Suite sans "l'installer". Il est ensuite simplement exécuté depuis le répertoire temporaire et ne modifie aucun paramètre spécifique à l'utilisateur.

Une nouvelle version peut notamment être exécutée à côté d'une ancienne version.

Installer

Le menu Outils/Installer permet l'installation d'une nouvelle version de SIARD Suite. Pour autant qu'une ancienne version ait été installée, cette dernière sera tout d'abord effacée.

Le menu Outils/Installer est désactivé si la version de l'instance actuellement exécutée de SIARD Suite est plus petite ou identique à la version de l'instance installée.

Dans le cas contraire, Outils/Installer exécute les tâches suivantes:

  • copiage des fichiers de l'instance actuellement exécutée dans un répertoire d'installation choisi par l'utilisateur,
  • création d'un fichier personnel avec les paramètres relatifs à la version et au répertoire de la nouvelle instance installée, et
  • création d'un raccourci sur le bureau pour l'exécution de SIARD Suite. Ceci n'est pas possible pour tous les systèmes d'exploitation, car par ex. les distributions LINUX utilisent un grand nombre de bureaux variables.

Désinstaller

Le menu Outils/Désinstaller élimine une version de SIARD Suite installée auparavant, pour autant qu'il en existe une.

Le menu Outils/Désinstaller est désactivé si aucun paramètre personnel n'a été trouvé.

Dans le cas contraire, Outils/Désinstaller exécute les tâches suivantes:

  • élimination du raccourci pour SIARD Suite du bureau, pour autant qu'il ait été trouvé,
  • effacement du répertoire d'installation et des fichiers de programmes sauf si l'instance actuellement exécutée a été lancée depuis là - ce répertoire doit alors être effacé manuellement plus tard, et
  • Effacement des paramètres personnels pour autant qu'il ait été requis.

Paramètres de SIARD

SIARD Suite lit et écrit ses paramètres depuis un fichier SIARD Suite_2.1.properties qui se trouve dans le répertoire de base .java de l'utilisateur.

Ce fichier de base de l'utilisateur est défini sur la valeur de la propriété system JAVA user.home. Pour autant qu'elle n'ait pas été redirigée manuellement par un argument JVM -D, cette propriété système a la même valeur que la variable d'environnement %USERPROFILE% sur la plate-forme Windows ou que la variable d'environnement $HOME (synonyme avec "~") sur les plates-formes LINUX/UNIX.

Normalement, on trouve ainsi les paramètres sous

  • C:\Users\<User>\.java\siard_suite_2.1.properties pour Windows,
  • /home/<User>/.java/siard_suite_2.1.properties pour la plupart des distributions LINUX.
