SIARD Suite 2.2 - Installation

Die folgenden Menüpunkte betreffen das Installieren:

  • Installieren: die aktuell ausgeführte Kopie von SIARD Suite installieren,
  • Deinstallieren: die aktuell installierte Kopie von SIARD Suite deinstallieren.

Zum Seitenanfang

Was bedeutet Installation?

SIARD Suite kann auf allen Plattformen genutzt werden, wo JAVA verfügbar ist. Deshalb ist die Installation nicht in einem separaten plattformspezifischen Installationsprogramm (z.B. setup.exe) realisiert sondern in SIARD Suite selber.

Installation bedeutet das Kopieren der Programmdateien z.B. von einem Wechseldatenträger oder einem anderen temporären Ort an einen festen Ort (Netzwerk oder Festplatte) und das Speichern einiger Einstellungen im persönlichen Verzeichnis des Benutzers, welche den Status vergangener Aufrufe des Programms festhalten (z.B.: C:\Users\<User>\.java\siard_suite-2.2.properties).

Es ist also möglich, SIARD Suite auszuführen, ohne es zu "installieren". Dann wird es einfach direkt vom temporären Ort ausgeführt und ändert keine benutzerspezifischen Einstellungen.

Insbesondere kann eine neue Version neben einer installierten alten Version ausgeführt werden.

Zum Seitenanfang

Installieren

Der Menüpunkt Extras/Installieren ermöglicht die Installation einer neuen Version der SIARD Suite. Sofern eine ältere Version installiert war, wird diese dabei vorgängig gelöscht.

Der Menüpunkt Extras/Installieren ist deaktiviert, wenn die Version der aktuell ausgeführten Instanz von SIARD Suite kleiner oder gleich der Version der installierten Instanz ist.

Andernfalls führt Extras/Installieren Folgendes aus:

  • Kopieren der Dateien der aktuell ausgeführten Instanz in einen vom Benutzer gewählten Installationsordner,
  • Erzeugen einer persönlichen Datei mit Einstellungen, welche die Version and den Ordner der neu installierten Instanz enthält, und
  • Anlegen einer Verknüpfung für das Ausführen von SIARD Suite auf dem Desktop. Dies ist nicht für alle Betriebssysteme möglich, da etwa LINUX-Distributionen eine grosse Vielzahl immer wieder ändernder Desktops nutzen.

Zum Seitenanfang

Deinstallieren

Der Menüpunkt Extras/Deinstallieren entfernt eine vorher installierte Version von SIARD Suite, sofern eine solche vorhanden ist.

Der Menüpunkt Extras/Deinstallieren ist deaktiviert, wenn keine persönlichen Einstellungen gefunden werden können.

Andernfalls führt Extras/Deinstallieren Folgendes aus:

  • Entfernen der Verknüpfung für SIARD Suite vom Desktop, sofern eine gefunden werden kann,
  • Löschen des Installationsordners mit den Programmdateien, ausser wenn die aktuell ausgeführte Instanz von dort aus gestartet wurde - dann muss dieser Ordner später manuell gelöscht werden, und
  • Löschen der persönlichen Einstellungen, sofern ihr Löschen gefordert wurde.

SIARD Einstellungen

SIARD Suite liest und schreibt seine Einstellungen von einer Datei siard_suite_2.2.properties, welche sich im Ordner .java im Home-Verzeichnis des Benutzers befindet.

Das Home-Verzeichnis des Benutzers wird über den Wert der JAVA Systemeigenschaft user.home bestimmt. Sofern dieser nicht manuell mit einem JVM -D Argument umgeleitet wurde, hat diese Systemeigenschaft denselben Wert wie die Umgebungsvariable %USERPROFILE% auf den Windows-Plattformen oder die Umgebungsvariable $HOME (synonym mit "~") auf LINUX/UNIX Plattformen.

Normalerweise findet man also die Einstellungen unter

  • C:\Users\<User>\.java\siard_suite_2.2.properties für Windows,
  • /home/<User>/.java/siard_suite_2.2.properties für die meisten LINUX-Distributionen.
