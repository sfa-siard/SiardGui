SIARD Suite 2.1 - Protokollierung

Wenn Probleme bei der Benutzung der SIARD Suite auftauchen, ist es nützlich, ein Ausführungsprotokoll zu speichern. Ein solches (oder - aus Sicherheitsgründen - eine Auszug daraus) sollte eine Fehlermeldung an die Wartung begleiten.

Um ein solches Protokoll aufzuzeichnen, geht man folgendermassen vor:

Im Verzeichnis etc der SIARD-Installation findet man die Datei logging.properties. In dieser Datei wird die Protokollierung mit Niveau WARNING auf die Konsole und alle Niveaus in die Logdatei %t/siard2%u.log umgeleitet. Das %t steht hier für den temporären Ordner (Wert der Umgebungsvariablen TEMP oder TMP) und das %u für eine Laufnummer. Unter Windows ergibt das normalerweise C:\Users\<user>\AppData\Local\Temp\siard20.log).

Das globale Logniveau steht anfänglich auf INFO. Ausführlicheres Logging kann eingestellt werden, indem man die Zeile
.level=INFO
in
.level=ALL
abändert. Derart ausführliches Logging verlangsamt die Ausführung.
