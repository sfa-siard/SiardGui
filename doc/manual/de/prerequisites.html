SIARD Suite 2.1 - Voraussetzungen

Voraussetzung für die Benutzung der SIARD Suite ist eine vorhandene Installation von JAVA. Voraussetzung zum Laden aus oder zu einem Datenbank-System ist eine entsprechende Infrastruktur.

Zum Seitenanfang

JAVA

Benutzer der SIARD Suite müssen vorgängig JAVA installieren. Die minimale technische Voraussetzung ist JRE 1.8.

JAVA ist frei erhältlich auf der Seite http://www.java.com/ (JRE – JAVA Runtime Environment) oder http://www.oracle.com/technetwork/java/javase/downloads/index.html (JDK - JAVA Development Kit). SIARD Suite nutzt JavaFX, welches Teil der JAVA SE Distribution, aber noch nicht im OpenJDK enthalten ist. Deshalb kann OpenJDK (http://openjdk.java.net/) nicht genutzt werden.

Ob JAVA 1.8 oder höher unter Windows verfügbar ist, kann man dies folgendermassen prüfen: Unter dem Menüpunkt "Start" den Befehl "cmd" eintippen und "java -version" im Befehlsfenster eingeben.

Architektur (32-bit/64-bit)

In früheren Versionen von SIARD Suite bestand eine Abhängigkeit zu ODBC. Dies hatte zur Folge, dass nur die 32-bit-Version von JAVA benutzt werden konnte, um auf MS Access Datenbanken zuzugreifen. SIARD Suite 2.1 benutzt ODBC nicht mehr für die Verbindung zu MS Access Datenbanken. Auf solche Datenbanken kann deshalb neu auch auf anderen Plattformen (z.B. LINUX) und mit 64-bit JAVA zugegriffen werden. Es wird empfohlen, die JAVA-Architektur entsprechend der Architektur des Betriebssystems zu wählen.

Zum Seitenanfang

Datenbanken

SIARD Suite 2.1 unterstützt zum Zeitpunkt dieser Auslieferung die folgenden Datenbanksysteme:

  • MS Access 2007 oder höher
  • DB/2 8 oder höher
  • H2 database 1.4 oder höher
  • MySQL (oder MariaDB) 5.5 oder höher
  • Oracle 10 oder höher
  • PostgreSQL 11 oder höher
  • SQL Server 2012 oder höher

Zu einem späteren Zeitpunkt können weitere Datenbanksysteme eingebunden werden. Die JDBC-Treiber der Datenbankhersteller sind normalerweise nicht SQL:2008-konform. Die meisten implementieren nicht einmal den JDBC 4 Standard auf den Gebieten der Datenbankmetadaten oder der komplexen Datentypen korrekt. Deshalb muss für jedes Datenbanksystem eine JDBC-Hülle entwickelt werden, welche wenigstens soweit konform zu den Standards it, wie dies von SIARD Suite benötigt wird.
