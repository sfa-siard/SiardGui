SIARD Suite 2.1 - Presupposti

Presupposto per l'utilizzazione di SIARD Suite è un'installazione JAVA disponibile. Il presupposto per scaricare da un sistema di banca dati o per caricare su di esso, è l'esistenza della relativa infrastruttura.

Inizio pagina

JAVA

Gli utilizzatori di SIARD Suite devono prima installare JAVA. Il presupposto tecnico minimo è JRE 1.8.

JAVA può essere scaricato liberamente dal sito http://www.java.com/ (JRE – JAVA Runtime Environment) oppure http://www.oracle.com/technetwork/java/javase/downloads/index.html (JDK - JAVA Development Kit). SIARD Suite utilizza JavaFX, che è parte della JAVA SE Distribution, ma non è ancora contenuta in OpenJDK. Per questo motivo OpenJDK (http://openjdk.java.net/) non può essere utilizzato.

Si può verificare se la versione JAVA 1.8 o superiore sia disponibile o meno sotto Windows, procedendo in questo modo: Sotto il punto di menu "Start" digitare il comando "cmd" e nella finestra di comando digitare "java -version".

Architettura (32-bit/64-bit)

Le precedenti versioni di SIARD Suite erano dipendenti da ODBC. Ciò significava che si poteva utilizzare solo la versione a 32-bit di JAVA, per accedere alle banche dati di MS Access. SIARD Suite 2.1 invece non usa più ODBC per la connessione alle banche dati di MS Access, e quindi si può accedere a queste anche da altre piattaforme informatiche (p.es. LINUX) e con la versione JAVA a 64-bit. Si consiglia di scegliere l'architettura JAVA in relazione all'architettura del sistema operativo.

Banche dati

SIARD Suite 2.1 supporta attualmente i seguenti sistemi di gestione di banche dati:

  • MS Access 2007 o superiore
  • DB/2 8 o superiore
  • H2 database 1.4 o superiore
  • MySQL (o MariaDB) 5.5 o superiore
  • Oracle 10 o superiore
  • SQL Server 2012 o superiore

In un secondo momento possono essere integrati altri sistemi di gestione di banche dati. I driver JDBC dei fornitori di banche dati di regola non sono conformi a SQL:2008. La maggior parte non integra neanche correttamente gli standard JDBC 4 nel campo dei metadati di banche dati o dei tipi complessi di dati. Per questo motivo, occorre sviluppare per ogni sistema di gestione di banche dati un rivestimento JDBC, che sia perlomeno conforme agli standard it, necessari a SIARD Suite.
