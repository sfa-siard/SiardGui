SIARD Suite 2.2 - Logging

Se sorgono problemi nell'utilizzo di SIARD Suite, è utile salvare un protocollo di esecuzione. Questo protocollo (oppure – per ragioni di sicurezza – un estratto di questo) dovrebbe essere allegato alla notifica d'errore da inviare all'ufficio manutenzione.

Per registrare questo protocollo, si proceda come segue:

Nella cartella etc dell'installazione SIARD si trova il file logging.properties. In questo file vengono inviate le registrazioni del protocollo con il livello WARNING alla console e deviati tutti i livelli nel file di logging %t/siard2%u.log. Il %t indica qui la cartella temporanea (valore della variabile TEMP o TMP) e %u il numero della numerazione. Sotto Windows si avrà normalmente C:\Users\<user>\AppData\Local\Temp\siard20.log).

Il livello globale di logging è impostato inizialmente su INFO. Un livello più approfondito del logging può essere impostato modificando la riga
.level=INFO
in
.level=ALL.
Un logging cosi dettagliato rallenta comunque l'esecuzione.
