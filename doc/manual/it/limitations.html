SIARD Suite 2.2 - Limitazioni

Il Format SIARD è sottoposto alle seguenti limitazioni:

  • Le dimensioni di un file SIARD non possono superare 18’446’744’073’709’551’615 bytes (ca. 18 ExaBytes, limitazione ZIP64).
  • Il numero dei file (tabelle e LOB) non può superare i 4'294'967’295 (ca. 4 miliardi) (limitazione ZIP64).

Queste limitazioni non dovrebbero rappresentare comunque in nessun caso degli ostacoli, dato che le banche dati reali non diventeranno così grandi ancora per molto tempo.

SIARD Suite è limitata inoltre da quanto segue:

  • L'insieme dei metadati della banca dati deve trovare posto nella memoria JAVA (Heap).

Questa limitazione può diventare effettiva realmente nel caso di una costellazione infelice di memoria principale disponibile e complessità dei metadati della banca dati. Si può evitare avviando SIARD su una macchina che dispone di memoria principale sufficiente (p.es. 4 GB) e che eventualmente aumenta il JAVA-Heap tramite il parametro della riga di comando JAVA: -Xmx2000m o simili sulla riga di comando JAVA.
