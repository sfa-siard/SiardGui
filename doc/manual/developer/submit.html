Submitting changes to the remote master branch

Whenever the changes in a development branch are stable, they can be submitted as a "pull request" to the release manager responsible for the master branch.

If the changes are approved, the release manager will pull the changes from the development branch (e.g. dev/enter) into the master branch running the following commands:

git pull origin dev/enter
git pull origin master
git push origin master

In addition the build is used to produce the release binaries. Those will have to be be published separately under "Releases" with the appropriate release number.
